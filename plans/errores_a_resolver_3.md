# Requisitos del Usuario - Errores a Resolver 3

## Prompt Original (preservado exactamente)

> ambas cosas pero la idea es que en cada modulo puedas elegir y visualizar todos los modelos disponibles que puedan ser utilizados para el proposito de ese modulo. lista primero los modulos, luego los modelos api y locales, luego para cada modulo debes hacer que aparezca en el mismo selector solo los modelos apis y locales que sirven para ese modulo, luego ve modulo y modelo viendo como la ui debe adaptarse a su caso

## Tareas Concretas

1. **Listar todos los módulos** de la aplicación
2. **Listar todos los modelos** (API y locales) disponibles
3. **Para cada módulo**: crear un selector unificado que muestre SOLO los modelos (API + locales) que sirven para ese módulo
4. **Para cada combinación módulo-modelo**: adaptar la UI a las opciones específicas de ese modelo

## Lo que NO debo hacer

- No simplificar ni resumir estos requisitos
- No asumir que algo ya está implementado sin verificar
- No hacer cambios parciales que no resuelvan el problema completo
